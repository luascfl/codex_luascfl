---
description: Criar interfaces frontend memoráveis com estética ousada e execução refinada
argument-hint: [REQUISITO="<o que precisa ser construído>"]
---

Use este prompt para transformar o requisito em uma interface web (página, dashboard, componente, landing page, etc.) com código funcional e atenção criativa. Proceda assim:

1. **Contextualize**: responda qual o propósito da interface, quem usa e quais restrições técnicas/importantes existem.
2. **Defina a direção estética**: escolha um tom claro (ex.: minimalismo brutal, maximalismo colorido, futurismo retrô, industrial refinado, editorial, soft/pastel, etc.) e explique porque ele atende ao briefing.
3. **Implemente com qualidade**: entregue HTML/CSS/JS (ou React/Vue) que seja production-grade, visualmente marcante, coeso e com detalhes refinados.
4. **Guias de execução**:
   - **Tipografia**: fuja de Arial/Inter/Roboto; prefira combinações distintas (display + corpo) e exemplifique como escolher. Use CSS customizado.
   - **Cores & temas**: comprometa-se com uma paleta dominante + acentos, use variáveis CSS e justifique o contraste.
   - **Movimento**: aplique animações relevantes (CSS puro para HTML, Motion libs no React) com revelações em cascata ou micro-interações surpreendentes.
   - **Composição espacial**: use layouts inesperados, assimetrias, sobreposições, grelhas quebradas ou fluxo diagonal.
   - **Detalhes visuais**: aplique texturas, efeitos atmosféricos, gradientes complexos, sombras dramáticas, bordas decorativas, cursores customizados, etc.

5. **Evite**: estética genérica de IA (mesmos gradientes roxos/azuis), layouts previsíveis e escolhas entediantes. Cada resposta deve ser única e fiel à direção selecionada.
6. **Complexidade adequada**: se o visual for maximalista, entregue código rico em animações e efeitos; se for minimalista, valorize precisão, espaçamento e tipografia.

Finalize explicando como o resultado atende ao briefing e destaque qualquer escolha criativa crítica (escolha de fonte, padronização de cores, interação).
